Game data

Each game session is identified by an server-unique table name you choose when you create it. A table name will be visible in the URL, but is not publically listed or shared by us. However, everyone who knows that name or can guess it, can join the table/session.

All data on a table - the pieces, the game board and everything else on it - is kept separate from other tables and other servers. Only those that have joined a particular table on a particular server can see/access that data.

Rooms do not store information about the players, only the current state of the table layout, the pieces and it's library / assets. Rooms also do not store the information mentioned above under Access data.

All data on a table is shared with all people on that table. Everyone at a table can download some or all this data at once (so-called snapshots) at any time.

Rooms and all their data are deleted from this server after a certain period of inactivity of that table. See the information below the join screen for the actual value of this period - usually 48 hours. If everyone leaves a table and nobody joins it again for that period, it and all contained game data & assets are automatically deleted.

After a table is deleted, it's name is free again for others to use. However, no data is taken over from the previous table with the same name.

Snapshots

A snapshot is a .zip file containing the state of a table at a certain point in time. This includes the game setup and all assets. Those that have joined a table can download such a snapshot to their local computer at any time.

While we delete tables after a period of inactivity (see above), we can not exert any control over downloaded snapshots by their previous players. They can recreate tables from those snapshots, either on this or any other FreeBeeGee server, by uploading them again at any time.
